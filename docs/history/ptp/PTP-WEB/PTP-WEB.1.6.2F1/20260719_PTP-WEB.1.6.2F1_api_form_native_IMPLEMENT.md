# PTP-WEB.1.6.2F1 — API e formulário comercial nativo

## Objetivo

Transformar o site PredixAI BR em uma base de validação comercial própria, sem Tally ou YayForms, usando Vercel e Supabase no plano gratuito.

## Arquitetura

```txt
Navegador
→ formulário nativo
→ POST /api/leads
→ RPC submit_commercial_lead
→ public.commercial_leads
```

## Segurança

- validação duplicada no navegador, API e banco;
- RLS forçada na tabela;
- leitura, alteração e exclusão públicas bloqueadas;
- inserção apenas pela função `SECURITY DEFINER`;
- rejeição de campos desconhecidos e dados sensíveis;
- honeypot;
- limite de cinco tentativas por hora por impressão técnica;
- sem chave administrativa no GitHub ou frontend.

## Produtos

```txt
PredixAI Atendimento = primeiro produto-base
PredixAI Pet = primeira vertical
PredixAI Market = pesquisa, sem promessa de produto pronto
```

## Páginas

- `/validacao/`
- `/privacidade/`
- `/obrigado/`
- `/solucoes/atendimento/`
- `/solucoes/pet/`
- `/solucoes/market/`

## Condição de avanço

A coleta pública só será liberada depois dos testes de API, RLS, antispam, consentimento, inserção válida, mobile, desktop e deployment de produção.

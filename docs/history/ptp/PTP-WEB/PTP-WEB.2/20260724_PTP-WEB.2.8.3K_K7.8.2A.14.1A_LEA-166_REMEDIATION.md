## 3. Achados recebidos

### F01 — conjunto exato de bindings não garantido

A implementação anterior comparava a quantidade de bindings encontrados com a quantidade de perguntas recebidas. Um documento parcial poderia, portanto, ser aceito.

### F02 — versão do schema de bindings não validada

A customização podia informar `bindingSchemaVersion` divergente da constante interna e ainda produzir configuração e hashes.

## 4. Correções

### Conjunto exato de bindings

Foi adicionada validação explícita entre:

```txt
EXPECTED_BINDINGS=ALLOWED_BINDINGS
ACTUAL_BINDINGS=seenBindings
```

O contrato agora rejeita:

- bindings obrigatórios ausentes;
- bindings desconhecidos;
- bindings duplicados;
- contagem diferente da allowlist completa de 19 itens.

### Binding schema version

A customização somente pode omitir a versão ou declarar exatamente `1.0.0`. Versões divergentes geram `BINDING_SCHEMA_VERSION_MISMATCH`. A saída usa sempre a constante interna validada.

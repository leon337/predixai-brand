# PTP-WEB.2.8.3K.7.8.2A.2 — Perguntas e respostas fictícias pré-preenchidas

## Objetivo

Criar o primeiro questionário adaptativo da **Clínica Aurora Diagnósticos**, oferecendo respostas fictícias prontas para que o usuário não comece com campos vazios e possa gerar um Atendente de IA próximo de uma operação real.

## Princípio central

```txt
EMPTY_START=FORBIDDEN
PRESELECTED_FICTIONAL_ANSWERS=ENABLED
USER_CAN_ACCEPT=YES
USER_CAN_EDIT=YES
USER_CAN_SKIP_OPTIONAL=YES
REAL_PATIENT_DATA=FORBIDDEN
```

## Ações disponíveis para cada resposta

```txt
1. Usar resposta sugerida
2. Editar resposta
3. Não incluir no teste
```

As proteções humanas permanecem obrigatórias e não são transformadas em opções removíveis.

## Seções iniciais do questionário

### 1. Identidade do negócio

- nome da clínica;
- descrição curta;
- região fictícia atendida;
- públicos simulados.

### 2. Identidade do funcionário

- nome do atendente;
- forma de apresentação;
- mensagem inicial.

### 3. Estilo de comunicação

- tom;
- tamanho das respostas;
- uso de emojis;
- frase de encaminhamento humano.

### 4. Contexto operacional

- canais fictícios;
- organização atual;
- volume simulado de solicitações;
- tipos de solicitação prioritários.

### 5. Informações autorizadas

- fontes fictícias publicadas;
- comportamento quando uma informação não estiver disponível.

### 6. Formato das respostas e registros

- confirmação do entendimento;
- perguntas objetivas;
- resultado;
- pendências;
- necessidade de aprovação humana;
- resumo do atendimento.

## Dependências preservadas

Esta etapa não publica detalhes de:

- catálogo de exames e serviços;
- horários e unidades;
- agendamento e cancelamento;
- pagamentos e convênios;
- perguntas frequentes completas;
- regras detalhadas de encaminhamento;
- cenários finais;
- prompt completo.

Esses conteúdos continuam reservados às etapas 3 a 10.

# 5. K.7.8.2A.14.1A — Contrato da Configuração Efetiva

```txt
LINEAR=LEA-166
STATUS=⬜ NÃO_INICIADA
```

## Objetivo

Definir o contrato determinístico entre pacote, personalização e configuração efetiva.

## Entregas

- allowlist de bindings;
- matriz `questionId → includeInPromptAs`;
- resolução de valor sugerido, editado e omitido;
- `rawValue`, `displayValue`, `promptInstruction` e `source`;
- mapa versionado de `promptInstruction` para o pacote da clínica;
- função pura `buildEffectiveAgentConfig()`;
- configuração imutável;
- serialização estável e hash.

# 8. K.7.8.2A.14.1D — Conflitos e Persistência

```txt
LINEAR=LEA-169
STATUS=⬜ NÃO_INICIADA
BLOCKED_BY=LEA-168
```

## Objetivo

Bloquear contradições e preservar a sessão sem criar fonte duplicada.

## Entregas

- detecção de conflitos de identidade;
- detecção de bindings inválidos;
- revisão explícita de texto editado manualmente;
- `bindingSchemaVersion=1.0`;
- recuperação via `sessionStorage` e `lastKnownGood`;
- hashes de configuração, personalização e prompt;
- `canonicalizationVersion=1`.

## Conflitos bloqueantes iniciais

```txt
employee.name × employee.initialMessage
company.displayName × employee.initialMessage
employee.presentation × employee.initialMessage
required × omitted
unknown option
unknown binding
canonical structure unknown
```

## Validações

```txt
IDENTITY_CONFLICT=BLOCK
MANUAL_TEXT_SILENT_REWRITE=NO
SESSION_RECOVERY=PASS
HASH_DETERMINISM=PASS
PROTOTYPE_POLLUTION=BLOCK
```

## Condição de avanço

Conflitos, migração mínima, hashes e recuperação da sessão em PASS.

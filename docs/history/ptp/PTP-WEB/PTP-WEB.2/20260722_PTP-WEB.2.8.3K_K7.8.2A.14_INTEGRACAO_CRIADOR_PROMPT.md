## Componentes implementados

### Cliente e validação

- `assets/js/workforce-package-client.js`
- consulta somente `/api/workforce-catalog`;
- valida pacote, versão, origem, estado, checksum e dez componentes;
- exige aviso visível para `github_build_fallback`;
- bloqueia geração em divergência de integridade.

### Adaptação

- `assets/js/workforce-package-adapter.js`
- adiciona o funcionário fictício Clara ao catálogo somente no modo pacote;
- converte questionário e cenários publicados para o contrato do criador;
- mantém os funcionários genéricos disponíveis fora desse modo.

### Estado temporário

- `assets/js/workforce-package-bootstrap.js`
- `assets/js/workforce-package-state.js`
- adiciona contexto de pacote e personalizações ao envelope da sessão;
- usa somente o namespace já governado em `sessionStorage`;
- invalida artefatos derivados quando versão ou checksum mudam;
- não grava personalizações no Supabase.

### Geração de prompt

- `assets/js/workforce-package-prompt.js`
- respostas sugeridas geram exatamente o prompt operacional canônico;
- alterações autorizadas geram uma seção determinística de personalização;
- perguntas obrigatórias não podem ser omitidas;
- limites e proibições do pacote continuam obrigatórios.

### Interface mobile-first

- `assets/js/workforce-package-ui.js`
- `assets/css/workforce-package.css`
- mostra aviso permanente de dados fictícios;
- oferece `Usar resposta sugerida`, `Editar resposta` e `Não incluir no teste`;
- divide o questionário em seções recolhíveis;
- preserva alvos de toque e layout responsivo.

## Correção encontrada durante o Preview

A primeira execução da API retornou `409 PACKAGE_INTEGRITY_FAILED`. A investigação confirmou que o checksum canônico existe na coluna `checksum_sha256`, enquanto o campo redundante `manifest.publicationGate.packageChecksum` não faz parte do manifesto publicado.

A correção manteve o comportamento fail-closed:

```txt
CANONICAL_ROW_CHECKSUM=REQUIRED
DUPLICATE_MANIFEST_CHECKSUM=OPTIONAL
IF_DUPLICATE_PRESENT_AND_DIFFERENT=BLOCK_409
SUPABASE_DATA_CHANGED=NO
SECURITY_REDUCED=NO
```

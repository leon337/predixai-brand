## Objetivo

Evoluir o site sem reconstrução para apresentar o PredixAI Workforce como produto carro-chefe, reorganizar módulos e verticais, expandir o diagnóstico comercial e preservar identidade, rotas, Vercel, Supabase e GitHub Pages.

## Decisões aprovadas

- Workforce como produto principal em validação;
- CTA principal “Solicitar diagnóstico”;
- Atendimento como primeiro departamento;
- Pet como vertical em validação;
- Market como pesquisa;
- soluções sob medida para OCR, visão, integrações e apoio à decisão;
- imagem conceitual acompanhada de explicação acessível em HTML;
- sem framework novo;
- GitHub Pages como espelho estático secundário;
- social card atualizado;
- controle humano explícito.

## Implementado na branch

- seção estática do Workforce após o Hero;
- página `/solucoes/workforce/`;
- departamentos com entradas, resultados, supervisão e estágio;
- diagrama conceitual acessível em SVG;
- reclassificação de Atendimento, Pet e Market;
- formulário com Workforce, Pet, Market e solução sob medida;
- novos detalhes para departamento, processos, canais, volume, integrações e supervisão;
- API sem fallback de configuração, com timeout e validação de origem;
- migração Supabase compatível com contratos antigos;
- SEO, JSON-LD, sitemap, manifest e social card;
- CSP, HSTS e cache de assets;
- build exigindo a entrega Workforce;
- GitHub Pages publicando apenas `dist/`;
- workflow de qualidade para PR;
- E2E sem criação automática de lead real em produção.


A RPC anônima permanece propositalmente executável porque é o endpoint público do formulário. A proteção depende de validação em duas camadas, rejeição de campos desconhecidos, consentimento, honeypot, rate limit e ausência de leitura pública. O aviso `RLS enabled no policy` representa a política deny-all intencional da tabela.

# PTP-WEB.2.8.2 — Refinamento Visual e Conversão

## Objetivo

Reduzir a altura visual da primeira tela comercial e do cabeçalho do diagnóstico para antecipar o benefício, o botão principal e a primeira pergunta.

## Origem

A validação visual da PTP-WEB.2.8.1 confirmou que a mensagem comercial estava clara, mas encontrou dois pontos de conversão:

1. o Hero ocupava altura excessiva no notebook;
2. o formulário demorava a apresentar a primeira pergunta no celular.

## Decisão

Ajustar apenas estilos responsivos, preservando:

- identidade visual;
- conteúdo comercial;
- arquitetura HTML;
- JavaScript;
- API;
- Supabase;
- rotas;
- produção atual.

## Arquivos alterados

```txt
assets/css/home-commercial.css
assets/css/commercial.css
```

## Alterações

### Home

- título reduzido em desktop;
- espaçamento superior e inferior reduzido;
- CTA antecipado;
- grade e painel visual compactados;
- espaço superior mobile reduzido;
- tipografia mobile recalibrada.

### Diagnóstico

- espaço entre cabeçalho e conteúdo reduzido;
- título reduzido no celular;
- descrição aproximada do título;
- formulário aproximado da primeira tela;
- cartão e título da etapa compactados em telas pequenas.

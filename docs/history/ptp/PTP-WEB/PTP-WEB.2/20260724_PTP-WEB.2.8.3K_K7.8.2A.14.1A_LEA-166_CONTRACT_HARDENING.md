## 4. Contrato de IDs

O questionário deve conter exatamente os 19 IDs esperados. As chaves de `answerModes`, `answers` e `omittedOptionalFields` devem pertencer ao mesmo conjunto.

Gates:

```txt
QUESTION_ID_SET=EXACT
UNKNOWN_CUSTOMIZATION_ID=REJECT
DUPLICATE_QUESTION_ID=REJECT
DUPLICATE_OMITTED_OPTIONAL_ID=REJECT
```

## 5. Respostas órfãs

Uma resposta personalizada somente pode existir quando o mesmo ID estiver em modo `edit_suggested`. Edições sem resposta e respostas associadas a modo sugerido ou omitido são rejeitadas.

```txt
ORPHAN_CUSTOMIZATION_ANSWER=REJECT
EDITED_ANSWER_REQUIRED=REJECT_WHEN_MISSING
OMISSION_MODE_CONFLICT=REJECT
```

## 6. Contrato do materializerMap

O mapa agora possui formato fechado:

```txt
packageId
version
instructions
```

Regras:

- `packageId` deve ser `health-medical-testing-clinic-aurora`;
- `version` deve ser `1`;
- `instructions` deve ser objeto próprio;
- deve possuir exatamente os 19 IDs do questionário;
- não pode possuir ID extra;
- toda instrução deve ser string não vazia.

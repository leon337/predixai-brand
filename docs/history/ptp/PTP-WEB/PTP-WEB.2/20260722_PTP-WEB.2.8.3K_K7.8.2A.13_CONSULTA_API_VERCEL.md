## Fallback versionado

```txt
SOURCE=github_build_fallback
COMPONENTS=10
FALLBACK_AVAILABLE=YES
WARNING=SUPABASE_UNAVAILABLE_USING_VERSIONED_BUILD_FALLBACK
SILENT_FALLBACK=NO
BOTH_SOURCES_UNAVAILABLE=NOT_READY_503
```

O fallback monta o pacote somente com arquivos versionados na mesma branch e valida identidade, checksum publicado, inventário, dez componentes e invariantes de segurança.

## Integridade fail-closed

Uma divergência de checksum, inventário, manifesto, aprovação ou invariantes na resposta do Supabase não pode ser mascarada pelo fallback.

```txt
PACKAGE_INTEGRITY_FAILURE=NOT_READY
HTTP_STATUS=409
FALLBACK_ON_INTEGRITY_FAILURE=BLOCKED
CACHE_CONTROL_ON_INTEGRITY_FAILURE=no-store
FAIL_CLOSED=PASS
```

O fallback é permitido apenas para indisponibilidade operacional ou ausência de configuração da fonte primária.

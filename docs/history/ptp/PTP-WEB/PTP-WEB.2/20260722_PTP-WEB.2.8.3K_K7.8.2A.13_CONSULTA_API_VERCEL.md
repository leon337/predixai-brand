```txt

HEALTH_QUERY=health=1

```

## Fontes de runtime

### Fonte primária

```txt
SOURCE=supabase_published_package
ACCESS=SUPABASE_PUBLISHABLE_KEY_SERVER_SIDE
RPC=public.get_published_workforce_catalog
RPC_MODE=SECURITY_DEFINER_READ_ONLY
```

A função SQL retorna somente pacotes `published` e `fictional=true`. A tabela continua protegida por RLS, sem permissão de leitura direta para `anon` ou `authenticated`.

O RPC foi validado com:

```txt

PAYLOAD_KEYS=10
RPC_ANON_EXECUTE=PASS
RPC_AUTHENTICATED_EXECUTE=PASS

```

### Fallback versionado

```txt
SOURCE=github_build_fallback
COMPONENTS=10
FALLBACK_AVAILABLE=YES
WARNING=SUPABASE_UNAVAILABLE_USING_VERSIONED_BUILD_FALLBACK
BOTH_SOURCES_UNAVAILABLE=NOT_READY
```

O fallback monta o pacote exclusivamente com os componentes versionados na mesma branch. Ele valida identidade, checksum publicado, inventário e invariantes de segurança antes de responder `READY`.

## Contrato de integridade

Antes de responder `READY`, a função exige:

- linha publicada e fictícia;
- pacote e versão presentes na allowlist;
- checksum SHA-256 aprovado;
- checksum do manifesto alinhado;
- dez componentes no payload;
- inventário integral esperado;
- aprovação registrada no manifesto;
- invariantes de segurança preservadas.

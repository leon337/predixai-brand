## 12. Conflitos bloqueantes

```txt

employee.presentation × employee.initialMessage
campo obrigatório × omissão
valor de opção inexistente
binding duplicado

estrutura canônica não reconhecida
```

Texto manual não será corrigido silenciosamente.

## 13. Persistência e migração mínima

Persistir somente:

```txt
answerModes
answers
omittedOptionalFields
updatedAt
bindingSchemaVersion=1.0
```

Não persistir `EffectiveAgentConfig` como fonte independente.

O artefato do prompt deverá registrar:

```txt
effectiveConfigHash
customizationHash
compilerVersion
packageChecksum
contentHash
```

Excluir dos hashes:

```txt
updatedAt
createdAt
sessionId
tabInstanceId
```

## 17. Riscos e controles

| Risco | Nível | Controle |
|---|---:|---|
| perda de regra canônica | alto | materialização por seções + paridade semântica |
| divergência entre UI e prompt | alto | mesma `EffectiveAgentConfig` |
| regressão do criador genérico | alto | projeção opcional + teste dedicado |
| alteração silenciosa de texto | médio | detectar e bloquear |
| hash não determinístico | médio | excluir dados temporais |
| quebra de sessão | médio | migração mínima + `lastKnownGood` |
| generalização prematura | médio | materializador específico da clínica |

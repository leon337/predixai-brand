## Validação JSON Schema

A primeira tentativa identificou incompatibilidade entre o uso de `prefixItems` com `items=false` e o comportamento do `pg_jsonschema 0.3.3`.

A correção aplicada foi:

```txt
sourcePriority.type=array
sourcePriority.minItems=2
sourcePriority.maxItems=2
sourcePriority.uniqueItems=true
sourcePriority.items.enum=[supabase_published_package, github_build_fallback]
```

A ordem continua protegida por validação SQL explícita. Após a correção:

```txt
SCHEMA_DEFINITION_VALID=PASS
MANIFEST_MATCHES_SCHEMA=PASS
SOURCE_PRIORITY_ORDER=PASS
```

## 4. Persistência e recuperação

Namespace:

```txt
predixai.employeeBuilder
```

Chaves conceituais:

```txt
ACTIVE_STATE_KEY
LAST_KNOWN_GOOD_STATE_KEY
PENDING_STATE_KEY
```

Fluxo atômico:

```txt
VALIDAR EM MEMÓRIA
→ GRAVAR PENDING
→ RELER E VALIDAR
→ PRESERVAR ACTIVE COMO LAST_KNOWN_GOOD
→ PROMOVER PENDING PARA ACTIVE
→ REMOVER PENDING
→ RELER E CONFIRMAR
```

A sessão é temporária, limitada à aba e nunca sincronizada. `sessionStorage` é tratado como entrada não confiável. Estado inválido não é executado como HTML ou código.

## 5. Prompt e teste externo

O prompt é um artefato com:

```txt
promptId
promptVersion
content
contentHash
dependencyHash
configurationVersion
humanControlVersion
authorizedContentVersion
catalogVersion

```

Regras adicionais são conteúdo não confiável. Instruções que tentam remover controles, ignorar limites ou inventar informações são rejeitadas.

A ponte externa permanece manual:

```txt
COPIAR PROMPT
→ ESCOLHER PLATAFORMA
→ SALVAR CHECKPOINT
→ ABRIR PLATAFORMA
→ RETORNAR À PREDIXAI
→ DECLARAR ENVIO
→ EXECUTAR CENÁRIO FICTÍCIO
→ REGISTRAR OBSERVAÇÃO
```

A PredixAI não lê a conversa externa, não confirma o envio e não valida a resposta. Toda observação é declarada pelo usuário.

## 10. K.7.5 — Critérios de aceitação e testes

### Jornada

- O usuário conclui `Criar → Conhecer → Testar → Decidir`.
- Nenhuma tela posterior abre sem guardas válidas.
- O retorno preserva respostas.
- A fase atual não depende apenas de cor.

### Segurança e prompt

- Controles obrigatórios não podem ser removidos.
- Regras conflitantes são rejeitadas.
- Cada prompt possui versão e hash.
- Cópia antiga não libera teste atual.
- O cenário só abre após declaração de envio.
- A PredixAI não lê a resposta externa.
- O usuário pode continuar sem testar.

### Recuperação

- Estado válido é salvo após transições.
- Falha de storage informa modo memória.
- Estado corrompido não é executado.
- `LAST_KNOWN_GOOD` permite recuperação segura.
- Exclusão remove apenas chaves do namespace.
- Sessão `UNKNOWN` não pode ser apagada.

### Prontidão e comercial

- A precedência é determinística.
- Capacidade humana é não elegível.
- Integrações exigem avaliação técnica.
- Conteúdo ausente gera pendência empresarial.
- Empresa é opcional.
- O usuário vê a prévia exata.
- Consentimento de contato é obrigatório.
- Novidades são opcionais.
- A mesma chave idempotente retorna a mesma referência.
- `UNKNOWN` bloqueia reenvio automático.

### Responsividade e acessibilidade

- Reflow funcional em 320, 360 e 390 CSS px.
- Zoom de 200% sem perda de função.
- Navegação por teclado.
- Foco muda para a nova tela.
- Erros usam `role=alert`.
- Status usa `role=status`.
- Movimento reduzido elimina animações não essenciais.
